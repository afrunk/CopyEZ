## CopyEZ

一个用于「快速收集、二次加工公文素材」的极简网页小工具，基于 Flask + SQLite 实现。

### 已完成功能

- **本地持久化存储**
  - 使用 SQLite（`copyez.db`）+ SQLAlchemy 存储素材，字段包括：标题、标签、正文内容、创建时间、最近编辑时间。
  - 启动时自动执行 `db.create_all()` 并通过 `ensure_schema()` 自修复缺失的 `updated_at` 字段，减少手动删库的需要。

- **首页素材列表（`/`）**
  - 扁平卡片列表展示所有素材，按创建时间倒序排列。
  - 每条卡片展示：标题、标签胶囊、时间等关键信息，支持鼠标悬停阴影提升可点击感。

- **新建素材页（`/new`）**
  - 表单字段：标题、标签（逗号分隔，可选）、内容。
  - 内容支持使用 `#` / `##` / `###` 作为一级/二级/三级标题标记，渲染时自动套用公文标题样式。
  - 服务端对内容做轻量换行归一化：统一换行符为 `\n`，并将连续空行压缩为单一空行，保证渲染结构稳定。
  - 必填校验：标题和内容为空时，在页面顶部展示错误提示并保留用户已输入的数据。

- **素材阅读页（`/note/<id>`）**
  - 使用 A4 公文版式进行排版，控制纸张宽度与页边距以接近真实打印效果。
  - **整篇文章总标题**使用「小标宋」风格，2 号字，居中显示，**不作首行缩进**；正文使用「仿宋」风格，约 16pt，1.75 左右行距，统一 2 字缩进。
  - **英文与数字排版说明**：在标题与正文的 `font-family` 中均优先使用西文字体（如 `Times New Roman`），仅在遇到中文字符或缺少字形时才回退到「小标宋」「仿宋」「楷体」等中文字库，因此英文与数字会尽量用英文专用字体渲染。
  - 支持在正文中渲染多级标题：
    - `#` 行 => 黑体一级标题。
    - `##` 行 => 楷体二级标题。
    - `###` 行 => 仿宋三级标题。
  - 使用 `render_content` 解析器，将用户原始文本（含 `#` 标记与换行）安全转换为 HTML，避免 XSS。
  - 页面右侧预留「随手笔记」侧栏区域，未来可用于提炼框架、重点标记等高级功能。
  - 顶部导航右侧提供「编辑」入口：
    - 使用扁平圆角按钮承载一个小钢笔图标（`icon-edit`），替代文字「编辑」，更精简现代。
    - 通过 `title` 与 `aria-label` 提供可访问性与悬停提示。

- **编辑已有素材页（`/edit/<id>`）**
  - GET：将已保存的标题、标签、正文内容「原样回填」到编辑表单，完整保留用户此前的 `#` 标题结构与换行。
  - POST：保存用户修改，同时更新 `updated_at` 为当前时间，保持与新建时一致的换行归一化逻辑。
  - 校验失败时（标题或内容为空），会在同一页面展示错误信息并保留用户当前输入。

- **统一的 UI 风格与交互细节**
  - 顶部应用栏：左侧 `CopyEZ` 标志，右侧圆形「+」图标按钮用于快速新建素材。
  - 主要按钮采用胶囊形态（主按钮 / 次按钮 / 文字按钮），并带有细腻的 hover 与 active 动画和阴影。
  - 标签使用莫兰迪色系胶囊风格，根据类型预留不同配色（如 `tag-full`、`tag-snippet`、`tag-framework`）。
  - 针对阅读页的打印场景做了专门优化：打印时隐藏顶部导航与侧栏，移除阴影，尽量还原 A4 纸张效果。
